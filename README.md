# SBU — Simple Backup Utility

Простой CLI-инструмент для резервного копирования файлов в S3-совместимое хранилище (MinIO) с поддержкой **дедупликации**.

## Возможности

- ✅ Настройка через YAML-конфиг
- ✅ Загрузка файлов в S3 (MinIO)
- ✅ Рекурсивное копирование директорий
- ✅ Дедупликация на уровне блоков (chunk-based deduplication)
- 🚧 Восстановление файлов из бэкапа
- 🚧 CLI-команды управления

## Как работает дедупликация

SBU использует **content-defined chunking** (библиотека `restic/chunker`):

1. Каждый файл разбивается на блоки переменного размера 
2. Для каждого блока вычисляется хеш SHA-256
3. Блоки сохраняются в S3 под именем `<sha256_hash>`
4. **Одинаковые блоки не дублируются** — если два файла содержат одинаковые данные, в хранилище будет только одна копия

Это позволяет экономить место при бэкапе похожих файлов (версии документов, VM-образы и т.д.).

## Требования

- Go 1.25+
- Docker (для запуска MinIO)

## Структура проекта

```
├── cmd/sbu/              # Точка входа
├── internal/
│   ├── config/           # Работа с конфигом
│   ├── storage/          # Загрузка чанков в S3 + дедупликация
│   ├── hash/             # Вычисление хешей файлов
│   └── backup/           # Копирование директорий
├── docker-compose.yml    # MinIO
└── config.yml            # Конфигурация (шаблон)
```

- [yaml.v3](https://gopkg.in/yaml.v3) — парсинг YAML-конфига
