# SBU — Simple Backup Utility

[![Go Version](https://img.shields.io/badge/Go-1.25+-blue.svg)](https://golang.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

Простой CLI-инструмент для резервного копирования файлов с поддержкой **дедупликации** и загрузкой в S3-совместимые хранилища (MinIO, AWS S3, Cloud.ru и другие).

## Возможности

- ✅ Настройка через YAML-конфиг
- ✅ Загрузка файлов в S3 
- ✅ Рекурсивное копирование директорий
- ✅ Дедупликация на уровне блоков (chunk-based deduplication)
- ✅ Восстановление файлов из бэкапа
- ✅ Просмотр списка снапшотов (`sls`)

## Как работает дедупликация

SBU использует **content-defined chunking** (библиотека `restic/chunker`):

1. Каждый файл разбивается на блоки переменного размера 
2. Для каждого блока вычисляется хеш SHA-256
3. Блоки сохраняются в S3 под именем `object/<sha256_hash>`
4. **Одинаковые блоки не дублируются** — если два файла содержат одинаковые данные, в хранилище будет только одна копия
5. Метаинформация (какие блоки принадлежат какому файлу) хранится в SQLite базе `sbu.db`

## Как работает восстановление

1. SBU скачивает базу данных `sbu.db` из S3 (содержит снапшоты)
2. По указанной пользователем дате и пути определяется:
   - **Файл** — скачиваются блоки в правильном порядке и собираются в файл
   - **Папка** — рекурсивно находятся все файлы внутри неё и восстанавливаются
3. Восстановленные файлы сохраняются по указанному пути

## Установка

```bash
git clone https://github.com/N-i-Kit-OS/sbu
cd sbu
go build -o sbu ./cmd/sbu

```
## Команды CLI

| Команда | Описание | Пример |
|---------|----------|--------|
| `init` | Создать пример конфига | `sbu init config.yml` |
| `run` | Запустить бэкап или восстановление | `sbu run config.yml` |
| `sls` | Показать список снапшотов из бакета (секция restore в конфиге) | `sbu sls config.yml` |

## Структура проекта

```bash
├── cmd/sbu/ # Точка входа (main.go)
├── internal/
│ ├── app/ # Оркестрация бэкапа/рестора/списка
│ ├── commands/ # CLI-команды (init, run, sls)
│ ├── config/ # Парсинг YAML-конфига
│ ├── storage/ # S3 клиент + чанкинг + дедупликация
│ ├── restore/ # Восстановление файлов
│ ├── sbudb/ # Работа с SQLite (снапшоты, файлы, блоки)
│ └── constants/ # Общие константы (префиксы, имена)
├── docker-compose.yml # MinIO для локального тестирования
└── config.yml # Пример конфигурации
```

| Библиотека | Назначение |
